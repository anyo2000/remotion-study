# V5 교차 검수

**모델**: opus (최종 판정)
**진행 표시**: `[5/6 검수완료]`

## 원칙
- **만든 놈 ≠ 검수하는 놈** — Claude가 만든 영상을 Gemini 3.0 Pro가 검수
- 검수 결과를 Claude가 **수용/무시/부분수용 판정 + 수정안**까지. 지적 리스트 나열로 끝내지 말 것
- mybrain 원본 데이터가 있으면 검수 모델에도 공유

## 할 일

### 5-1. 자동 검사 (기계적, Claude 직접)
1. **폰트 크기 검증**: `node scripts/check-fontsize.js` 실행
   - 52px 미만 발견 시 → 즉시 수정 (사용자에게 보여주기 전에)
   - 동적 fontSize(변수, 계산식)도 최솟값 52 이상 확인
2. **팔레트 준수**: .tsx 파일에서 하드코딩 색상 grep → constants.ts 팔레트 외 색상 없는지
3. **중앙 정렬**: 레이아웃이 중앙축에서 벗어나지 않는지

### 5-2. 스틸 캡처
- 각 장면의 애니메이션 완료 시점에서 스틸 캡처:
  ```
  npx remotion still src/index.ts {CompositionName} out/still-{N}.png --frame={frame}
  ```

### 5-3. Gemini 교차 검수 (핵심)
- 스틸 이미지를 Gemini 3.0 Pro에 전달:
  ```
  python3 scripts/cross_review.py out/still-1.png out/still-2.png out/still-3.png ...
  ```
- Gemini가 검수하는 항목:
  - 텍스트 가독성 / 크기
  - 레이아웃 균형 / 중앙 정렬
  - 색상 일관성 / 팔레트 준수
  - 정보 전달력 / 핵심 메시지 집중도
  - 여러 장일 때: 프레임 간 시각적 일관성

### 5-4. 오디오 싱크 점검 (Claude 직접)
- 텍스트 등장 타이밍 ≥ 나레이션 해당 부분 시작 시간
- 음성보다 먼저 정보 노출 금지

### 5-5. Claude 판정 (자동)
Gemini 검수 결과의 각 issue에 대해:

| 판정 | 의미 | 후속 |
|------|------|------|
| **수용** | Gemini 지적이 맞음 | 즉시 코드 수정 |
| **부분수용** | 문제는 맞지만 다른 방향으로 | 다른 수정안 적용 |
| **무시** | 컨텍스트 부족 or 의도된 것 | 이유 기록 |

- 수용/부분수용 건은 수정 후 해당 장면만 다시 스틸 캡처
- 전부 무시가 아닌 이상, 수정 결과를 반영한 최종본 기준으로 보고

### 5-6. 사용자에게 제시
- 스틸 이미지 (수정 후 최종본)
- 자동 검사 결과
- Gemini 지적 + Claude 판정 요약
- 사용자 확인 대기
