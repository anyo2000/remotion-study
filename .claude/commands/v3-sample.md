## 할 일

### 3-1. 장면 쪼개기
대본을 읽고 자연스러운 장면 경계를 정한다. 음성 타임스탬프 기반.

### 3-2. 설계 원칙 체크 (방향표 작성 전 반드시 확인)

**주인공은 하나.** 나머지는 주인공을 돕는 역할. 아래 체크리스트를 장면마다 적용:

- [ ] **밀도 관리** — 요소 4개 이상 세로 나열이면 좌우 분할 고려
- [ ] **억지 대비 금지** — 비교가 핵심 메시지가 아니면 한쪽만 크게 보여줄 것. "대비를 위한 대비" 하지 말 것
- [ ] **등장 순서 = 시각적 위계** — 핵심(큰 글씨) 먼저 등장, 보조(박스/카드) 나중. 빈 박스가 먼저 보이면 안 됨
- [ ] **숫자+단위 한 줄** — "200개"가 "200\n개"로 되는 레이아웃이면 폰트 축소 or 박스 확대

### 3-3. 장면별 비주얼 방향표 작성

**각 장면마다 아래 형식으로 제안:**

```
┌─ 씬 N: {제목} ({시작초}~{끝초}) ──────────────┐
│ 감정: {시청자가 느껴야 할 것}                      │
│ 주인공: {비주얼 주인공 — 이모지? 숫자? 차트?}       │
│ 접근: {구체적 비주얼 방식}                         │
│ 레이아웃: {중앙 집중 / 좌우 분할 / 산개 / 그리드}   │
│ 참조: {후킹와이 어떤 장면 느낌인지}                  │
│ 질문: {결정 안 된 것 — 사용자에게 물어볼 것}         │
└──────────────────────────────────────────────┘
```

### 3-4. 사용자 피드백 수렴
- 사용자가 "A안/B안" 선택하거나 "이건 아닌데" 하면 수정
- **모든 장면의 비주얼 방향이 확정될 때까지 v4로 넘어가지 않는다**

### 3-5. (선택) 미니샘플
사용자가 "이건 샘플 보여줘"라고 하면 해당 장면만 빠르게 코딩해서 스틸 제시.
사용자가 요청하지 않으면 건너뜀 — 비주얼 방향 합의만으로 v4 진행 가능.

## 관문 조건
- 전 장면의 비주얼 방향이 사용자에게 확인됨
- 명시적 승인("좋아", "진행해", "ㅇㅇ") 없이 v4로 절대 넘어가지 않음
