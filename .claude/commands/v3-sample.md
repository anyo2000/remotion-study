# V3 미니샘플 (핵심 관문)

**모델**: opus (비주얼 판단)
**진행 표시**: `[3/6 샘플승인]`

## !! 이 단계는 절대 건너뛰지 않는다
2026-04-10 실패: 미니샘플 없이 9개 장면 전체 코딩 → "지루하다/유치하다/크기 안 맞다" → 전면 수정.
비용: 시간 3배, 토큰 낭비. 미니샘플 2개면 방향 확인 가능.

## 할 일

1. **대표 장면 1~2개 선정** (시각적으로 가장 복잡한 장면)
2. **5색 팔레트 확정**: 배경 / 카드 / 본문텍스트 / 보조텍스트 / 강조
3. **Tight 스펙 작성** (프레임 단위, spec-writing 스킬 참조)
4. **미니샘플 제작** (5초, Remotion Composition으로)
5. **스틸 캡처**: `npx remotion still` 로 핵심 순간 캡처
6. 사용자에게 제시:
   - 스틸 이미지
   - 애니메이션 설명
   - 팔레트 스와치

## 사전 참조 (반드시 로드)
- remotion-best-practices 스킬 (spring 설정, 안티패턴)
- component-registry 스킬 (기존 컴포넌트 먼저 확인)
- spec-writing 스킬 (tight 스펙 형식)

## 사용자 확인 항목
- 배경 스타일
- 색상 팔레트
- 폰트 크기와 레이아웃 그리드
- 장식 요소 (또는 없음)
- 애니메이션 느낌 (bouncy / smooth / dramatic)

## 관문 조건
- 사용자가 OK / 좋아 / 이대로 / 괜찮아 → 승인
- 피드백 있으면 → 수정 → 다시 제시
- **명시적 승인 없이 v4로 절대 넘어가지 않는다**
