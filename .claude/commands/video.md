# 영상 제작 오케스트레이터

사용자가 영상 제작을 요청하면 이 파이프라인을 따른다.

## 시작 메시지
이 스킬이 로드되면 가장 먼저 다음 메시지를 출력한다:
> 🔄 video 오케스트레이터 시작

## 파이프라인 단계

| 단계 | 명령 | 모델 | 관문 | 역할 |
|------|------|------|------|------|
| 0 | `/v0-intake` | opus | - | 주제·소재·형식 파악 |
| 1 | `/v1-concept` | opus | 사용자 선택 | 교육 합의 + 컨셉 3안 + 비주얼 방향 |
| 2 | `/v2-script` | sonnet | 대본 승인 | 풀대본 + TTS + 타임스탬프 |
| 3 | `/v3-sample` | opus | **반드시 승인** | 장면별 비주얼 방향표 + (선택)미니샘플 |
| 4 | `/v4-build` | sonnet | - | beats.ts + 전체 씬 코딩 |
| 5 | `/v5-qa` | opus | 사용자 확인 | 자동검사 + Gemini 교차검수 |
| 6 | `/v6-render` | sonnet | - | MP4 렌더 + Drive + git |

## 자동 체이닝
각 단계가 완료(승인 포함)되면 다음 단계 스킬을 자동 호출한다.
관문 단계에서는 사용자 승인 전까지 다음으로 넘어가지 않는다.

## 형식 감지
- 1.5분 이하 → 티저 (세로 9:16, 단일 패스)
- 1.5분 초과 → 교육영상 (가로 16:9, 파트 분할 2~3분씩)
  - 각 파트가 v3~v4를 독립 수행

## 시리즈 연속편 단축
시리즈 연속편(같은 시리즈 폴더에 이전 에피소드 존재)은 v1을 스킵한다:
- 대본 확정 → v0 → v1(스킵) → v3 → v4 → v5 → v6
- 대본 미확정 → v0 → v1(스킵) → v2 → v3 → v4 → v5 → v6

## 핵심 규칙
- **v3(비주얼 방향)은 절대 건너뛰지 않는다.** 방향 합의 없이 코딩하면 전면 수정
- 각 단계 완료 시 진행 표시를 메시지 끝에 붙인다
- 사용자가 주제+소재를 한 번에 주면 v0+v1 압축 가능

## 진행 표시 포맷
```
[N/6 상태] → 다음: 다음단계명
```

## 모델 라우팅
- **opus**: 창작/판단이 필요한 단계 (v0, v1, v3, v5)
- **sonnet**: 기계적 실행 단계 (v2, v4, v6)

지금부터 v0-intake를 시작한다.

## 완료 메시지
이 단계의 모든 작업이 끝나면 반드시 다음 메시지를 출력한다:
> ✅ video 오케스트레이터 완료

---✅ video 스킬 실행
