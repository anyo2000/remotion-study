# 영상 제작 오케스트레이터

사용자가 영상 제작을 요청하면 이 파이프라인을 따른다.

## 파이프라인 단계

| 단계 | 명령 | 모델 | 관문 |
|------|------|------|------|
| 0 | `/v0-intake` | opus | - |
| 1 | `/v1-concept` | opus | 사용자 선택 |
| 2 | `/v2-script` | sonnet | 대본 승인 |
| 3 | `/v3-sample` | opus | **반드시 승인** |
| 4 | `/v4-build` | sonnet | - |
| 5 | `/v5-qa` | opus | 사용자 확인 |
| 6 | `/v6-render` | sonnet | - |

## 형식 감지
- 1.5분 이하 → 티저 (세로 9:16, 단일 패스)
- 1.5분 초과 → 교육영상 (가로 16:9, 파트 분할 2~3분씩)
  - 각 파트가 v3~v4를 독립 수행

## 핵심 규칙
- **v3(미니샘플)은 절대 건너뛰지 않는다.** 2026-04-10 전면 수정 실패 사례 참고
- 각 단계 완료 시 진행 표시를 메시지 끝에 붙인다
- 관문 단계에서는 사용자 승인 전까지 다음으로 넘어가지 않는다
- 사용자가 주제+소재를 한 번에 주면 v0+v1 압축 가능

## 진행 표시 포맷
```
[N/6 상태] → 다음: 다음단계명
```
교육영상 파트 분할 시:
```
[N/6 상태 P2/3] → 다음: 다음단계명
```

## 모델 라우팅
- **opus**: 창작/판단이 필요한 단계 (v0 접수, v1 컨셉, v3 샘플, v5 검수)
- **sonnet**: 기계적 실행 단계 (v2 TTS, v4 코딩, v6 렌더링)
- sonnet 단계는 Agent 서브에이전트를 `model: "sonnet"`으로 호출하여 처리

지금부터 v0-intake를 시작한다.
