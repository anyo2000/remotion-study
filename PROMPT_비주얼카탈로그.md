# 비주얼 카탈로그 제작 요청

## 배경
나는 네가 Remotion으로 뭘 만들 수 있는지 머릿속에 그림이 없어.
이전에 hooking-why 만들 때 네가 3안 제시하면 내가 골랐는데, 렌더링 결과가 내 상상과 달라서 v1→v2→최종까지 반복했잖아. subwayframe, chainreaction, arrowfocus 같은 창의적 시도가 렌더링하면 이상했고, 결국 keyword, comparison, dialogue 같은 단순 타입으로 돌아왔어.

근본 원인: **내가 각 타입이 실제로 어떻게 보이는지 모르니까 선택을 못 하는 거야.**

## 요청
**비주얼 샘플 카탈로그**를 만들어줘. 하나의 Composition으로 렌더링해서 내가 쭉 보면서 "아 이런 느낌이구나, 저건 좋다, 이건 별로" 판단할 수 있게.

## 구성 — 3개 축

### 축 1: 같은 내용 → 다른 씬 타입
하나의 대본 문장을 정해서, 기존 7종 씬 타입으로 각각 표현해줘.
- [키워드 강조], [비교], [흐름도], [카드 나열], [숫자 임팩트], [대화 UI], [반전]
- 7종 전부 안 어울리는 문장도 있을 테니, 억지로 다 만들지 말고 자연스러운 것만.
- 대본 문장은 네가 골라. 조건: **다양한 타입이 어울리는 문장**으로. 보험 교육 맥락이면 좋고.

### 축 2: 같은 씬 타입 → 다른 모션/등장 방식
keyword 타입 하나를 기준으로, 같은 텍스트+이모지인데 등장 방식만 다르게:
- fade in, slide up, pop in, type in (타자기), scale up, blur→clear 등
- spec-writing 스킬에 있는 14가지 비주얼 방향 용어 중에서 실제로 구현 가능한 것들을 보여줘.
- 추가로 네가 Remotion spring()이나 interpolate()로 할 수 있는 재밌는 등장 방식이 있으면 포함해.

### 축 3: 같은 씬 타입 → 다른 톤/레이아웃
comparison이나 card 타입으로:
- 정돈 톤 (컨테이너 + 라인 아이콘 + 텍스트 위계)
- 가벼운 톤 (이모지 + 컬러풀 + 큰 도형)
- 같은 정보인데 톤만 바꾸면 느낌이 얼마나 다른지 보여줘.

## 규칙
- 각 샘플의 길이는 자유. 효과를 충분히 보여줄 수 있는 만큼. 2초든 5초든.
- 샘플 사이에 "축1-키워드강조", "축2-slide up" 같은 라벨 카드를 넣어서 뭘 보는 건지 알 수 있게.
- 기존 CLAUDE.md 규칙 (52px 최소, 팔레트, 세이프존, 중앙축 정렬) 전부 적용.
- 팔레트는 하나로 통일 (카탈로그니까 색 변수를 없애야 비교가 됨).
- 음성/TTS 불필요. 비주얼만.
- 가로(16:9) 기준.

## 목적
이 카탈로그를 본 다음부터, 새 영상 스토리보드를 짤 때 "씬 4는 카탈로그 축1의 3번째 느낌으로" 이렇게 소통하려고 해. 우리 사이의 **공유 비주얼 사전**을 만드는 작업이야.

## 추가
- 네가 보기에 위 3개 축 외에 "이것도 보여드려야 차이를 느낄 수 있다" 싶은 게 있으면 축을 추가해도 돼.
- 만들기 전에 어떤 샘플들을 넣을 건지 목록을 먼저 보여주고, 내가 확인한 다음 코딩 시작해.

## 진행 방식 — 한 번에 제대로

이 작업은 설계를 잘못 잡으면 중간에 엎어야 하니까, 단계를 나눠서 진행해.

### 1단계: 플랜 (plan mode)
플랜 모드로 먼저 설계해.
- 전체 Composition 구조 (Sequence 배치, 라벨 카드 포함 총 길이)
- 축별로 어떤 샘플을 넣을지 전체 목록 (샘플 이름, 예상 길이, 사용할 컴포넌트)
- 기존 컴포넌트 중 재사용할 것 vs 새로 만들어야 할 것 분류
- 팔레트 확정
- 내가 확인하고 승인하면 다음 단계로

### 2단계: 코딩 (ultrathink)
승인된 플랜 기반으로 ultrathink로 코딩해.
- 한 축씩 만들고 중간 렌더링으로 확인하면서 진행
- 새 컴포넌트 만들 때는 기존 컴포넌트(GradientBackground, SceneHeadline, FadeInText 등) 최대한 재사용
- 각 샘플이 독립적으로 작동하게 (나중에 개별 샘플만 뽑아서 레퍼런스로 쓸 수 있도록)

### 3단계: 렌더링 + 리뷰
- 전체 카탈로그 영상 렌더링
- 내가 보고 "이건 좋다 / 이건 별로" 피드백
- 좋다고 한 것들은 references/에 개별 레퍼런스로 등록
