# Remotion 영상 제작 원칙

## 워크플로우
- 대본 먼저 → TTS 음성 생성 → 음성 길이에 맞춰 영상 제작
- 장면 사이 0.7초 쉼
- 장면 전환은 TransitionSeries fade 사용

## 텍스트
- 최소 폰트 48px (모바일 가독성)
- 설명은 키워드만 — 문장 전체를 화면에 넣지 않음
- 화법(고객에게 할 말)은 전체 문장 그대로 보여줌
- 부가 설명 텍스트 넣지 않음

## 색상
- 기본: 흰색(#f0f0f0) + 연한 블루(#7cb3e0) + 회색(#6b7280)
- 강조 오렌지(#e8943a)는 영상 전체에서 1~2곳, 진짜 핵심만
- 원색 남발 금지

## 레이아웃
- Safe Zone: 상단 150px, 하단 170px, 좌우 60px
- 세로(9:16, 1080x1920) 기준 위→아래 흐름
- 빈 공간은 요소를 키워서 채움 (억지 장식 X)
- 배경에 radial-gradient 깊이감 적용

## 애니메이션
- 의미 없는 반복 애니메이션 금지
- 등장 타이밍은 음성과 맞춤 — 음성보다 먼저 정보 노출 금지
- spring 기반 자연스러운 움직임
- 숫자 카운트업에 fontVariantNumeric: "tabular-nums"

## 장면 유형 (대본에 태그로 표시)
- [키워드 강조] — 핵심 단어 크게
- [비교] — 막대 차트, A vs B
- [흐름도] — 단계별 화살표 연결
- [카드 나열] — 항목 리스트
- [숫자 임팩트] — 카운트업
- [대화 UI] — 말풍선 (화법용)

## TTS
- OpenAI tts-1-hd, voice: nova, speed: 1.4
- 음성 파일은 public/audio/에 저장
- Pretendard 폰트 사용 (CDN)
