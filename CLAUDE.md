# Remotion 영상 제작 원칙

## 워크플로우 — 짧은 영상 (1분 내외 티저)

### 제작 흐름 (오디오 퍼스트)
```
① 메시지 + 소재 지정 — 차장님이 주제와 참고 소재 위치를 알려줌
② 키메시지 + 컨셉 3안 — 표현 방향 + 톤 제안. 차장님이 하나 선택
③ 풀대본 작성 → TTS 생성 → 전 장면 키워드별 세부 타임스탬프 한번에 추출
④ 장면별 스토리보드 — 핵심 장면 5초 미니샘플로 시각 컨셉 확인
⑤ [Plan Mode] 전체 구조 확정 → 제작 → 디테일 조절
⑥ 렌더링(MP4) → Google Drive 저장 → git 커밋
```

### ⚠️ ④번 관문 — 절대 건너뛰지 말 것
- ③ 끝나면 **반드시 ④에서 멈출 것**. 바로 ⑤로 가지 말 것
- **대표 장면 1~2개를 미니샘플(5초)로 먼저 만들어서** 차장님에게 확인받기
- 이 단계에서 확정할 것: 배경 스타일, 색감, 글자 크기, 레이아웃 그리드, 장식 요소
- 차장님이 OK 해야 나머지 장면 확장
- ③에서 세부 타임스탬프를 전 장면 한번에 뽑아야 ④에서 싱크 맞는 미니샘플 가능
- **2026-04-10 실패**: ④ 건너뛰고 9개 장면 전체 코딩 → "지루하다/유치하다/크기 안 맞다" 전면 수정 반복

### 최종 저장 경로
- **Google Drive**: `~/Library/CloudStorage/GoogleDrive-anyo2000@gmail.com/내 드라이브/즐겨찾아/클로드작업/영상/{제목} {YYMMDD}.mp4`
- 차장님이 "저장하자" / "이대로" 하면 렌더링 + Drive 저장까지 포함해서 진행할 것
- 대본은 컨셉 확정 후, 코딩 전에 완성. TTS 음성 길이가 각 장면의 프레임을 결정
- 모션 톤은 "이 정도 느낌"만 컨셉에서 잡고, 세부 조절은 제작 후 후조정
- 5분 이상 긴 영상은 이 흐름이 아닌 별도 방식 필요 (파트 분할 등)
- **장면 스펙 구체성 기준**: `references/prompt-quality-standards.md` 참조. 코딩 전에 장면 목적/텍스트/레이아웃/애니메이션/타이밍/톤이 모두 정해져야 함. 모호하면 차장님에게 선택지 제시

### 작업 전 필수 참조
- **티저 대본 작성 시** → 프로젝트 메모리 `reference_teaser_script_guide.md` 읽고 시작할 것

### 제작 원칙
- **셀프 검수**: 코딩 완료 후 각 장면의 애니메이션이 끝난 시점에서 스틸컷(`npx remotion still`)을 찍어 직접 확인. 폰트 크기, 색상, 레이아웃, 정보 배치 문제를 차장님한테 보여주기 전에 먼저 잡을 것
- 교안은 구조/맥락 이해용 — 비교인지, 흐름인지, 강조인지 판단하는 데만 사용
- 교안의 도형, 레이아웃, 색상을 따라 그리지 않음. 모션그래픽으로 자유롭게 재해석
- 텍스트보다 움직임, 도형, 아이콘으로 개념을 표현할 것
- 장면 사이 0.7초 쉼
- 장면 전환은 TransitionSeries fade 사용
- 마지막 장면 끝에 반드시 2~3초(60~90프레임) 여유 추가 — 음성이 잘리지 않도록

## 텍스트
- 최소 폰트 52px — 이보다 작은 글씨는 어떤 장면에도 절대 사용 금지
- 설명은 키워드만 — 문장 전체를 화면에 넣지 않음
- 화법(고객에게 할 말)은 전체 문장 그대로 보여줌
- 부가 설명 텍스트 넣지 않음
- **두괄식 헤드라인**: 장면 시작 시 상단에 핵심 키워드 배치 (예: "만남을 꺼림", "의리가입 종말")
  - 시청자가 "지금 뭘 말하는지" 바로 알 수 있게
  - 예외: 대본에 명확한 반전 구조([반전] 태그)가 있을 때만 헤드라인 생략

## 색상 — 팔레트 시스템
- **매 영상 시작 전 5색 팔레트를 먼저 확정**하고 코딩 시작
  - 배경 / 카드·박스 / 본문 텍스트 / 보조 텍스트 / 강조 (1색)
- 전 장면 동일 팔레트 유지 — 장면마다 색이 바뀌면 안 됨
- 팔레트에 없는 색은 사용 금지 (즉흥으로 색 넣지 말 것)
- 강조색은 영상 전체에서 1~2곳, 진짜 핵심만
- 변화가 필요하면 같은 색의 명도(tint/shade)로 구분 — 새로운 색 추가 금지
- 원색 남발 금지

- **가로(16:9, 1920×1080)**: Safe Zone 상하 80px, 좌우 100px. 좌→우 흐름

- 의미 없는 반복 애니메이션 금지
- 등장 타이밍은 음성과 맞춤 — 음성보다 먼저 정보 노출 금지
- spring 기반 자연스러운 움직임
- 숫자 카운트업에 fontVariantNumeric: "tabular-nums"

## 장면 유형 (대본에 태그로 표시)
- [키워드 강조] — 핵심 단어 크게
- [비교] — 막대 차트, A vs B
- [흐름도] — 단계별 화살표 연결
- [카드 나열] — 항목 리스트
- [숫자 임팩트] — 카운트업
- [대화 UI] — 말풍선 (화법용)
- [반전] — 두괄식 헤드라인 생략, 전환점에서 키워드 등장

## 장면 톤 — 무거움/가벼움 구분
- **정돈 톤** (정의, 화법, 데이터, 핵심 전달): 글자 + 박스 + 차트 + 막대그래프 위주. 신뢰감, 공식 자료 느낌.
- **가벼운 톤** (사례, 에피소드, 공감, 분위기): 이모지 + 도형 + 움직임 위주. 직관적, 눈길 끔.
- 한 영상 안에서 섞어 쓸 수 있음 — 대본의 맥락에 따라 자동 판단
- 판단 기준: "이걸 외워야 하나?" → 정돈 톤 / "이건 느끼면 되나?" → 가벼운 톤

### 정돈 톤 참고 기준 (references/style-clean/ 참조)
다음은 정돈 톤 장면에서 **참고할 방향**이다. 제약사항이 아니라 분위기 가이드.
- **구조적 정보에 컨테이너 활용 권장** — 프로세스/단계/비교 장면에서 라운드 박스, 원형 테두리 등으로 요소를 감싸면 정돈감 올라감. 단순 텍스트 장면에는 불필요
- **아이콘 vs 이모지는 상황별 판단** — 라인 아이콘이 어울리면 쓰고, 못생기면 이모지가 나음. 감정/상황 묘사에는 이모지가 더 어울릴 수 있음
- **텍스트 크기 위계 명확하게** — 제목/부제/강조/캡션 사이 크기 차이를 충분히 줄 것
- **중앙축 정렬 철저** — 절대 규칙

## TTS
- **Gemini Pro** (`gemini-2.5-pro-preview-tts`) — 품질 우선. 스타일 프롬프트로 톤 제어
- 음성 카탈로그: `references/gemini-voice-catalog.md` (8종, 용도별 추천)

- 음성 파일은 `public/audio/`에 저장

## 오디오-영상 싱크

- 파이프라인: TTS 생성 → Whisper 타임스탬프 추출 → 키워드 프레임 확인 → Sequence from 값 설정

## 레퍼런스 관리

### 규칙
1. 차장님이 참고 영상, 스타일 샘플, 모션 레퍼런스를 주면:
   - `references/` 폴더에 저장 (영상 링크는 `references/index.md`에 기록)
   - `references/index.md`에 뭔지, 어떤 점을 참고할지 한줄 기록
2. 새 영상 작업 시 `references/index.md`를 먼저 확인
3. "이 움직임 좋아", "이건 너무 화려해" 피드백 → 이 CLAUDE.md의 해당 섹션에 누적 반영
4. 대본/교안 원천 데이터는 mybrain 레포에서 가져옴 (~/Desktop/workspace/mybrain/)
   - `knowledge/` — 월별 회의자료, 상품 지식 (마크다운 정리본)
   - `knowledge/index.md` — 전체 지식 목차 (여기서 먼저 뭐가 있는지 확인)
   - `output/` — 과거 교육 결과물 (PPT, 대본) → 톤/스타일/구성 참고용
