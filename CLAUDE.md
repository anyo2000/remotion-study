# Remotion 영상 제작 원칙

## 파이프라인
영상 제작은 6단계 스킬로 자동 진행된다. "영상 만들자" → AI가 순차 실행.
개별 단계 점프: `/v0-intake` `/v1-concept` `/v2-script` `/v3-sample` `/v4-build` `/v5-qa` `/v6-render`
오케스트레이터: `/video` — 각 단계의 모델(opus/sonnet), 관문, 진행 표시 규칙 포함.

## 역할
- 차장님: 방향 결정, 판단, 승인 (director)
- Claude: 구체화, 제작, 검수 (producer + builder)
- 느낌/방향 중심 피드백 → Claude가 구체적 수치로 변환

## 텍스트
- 최소 폰트 52px — 어떤 장면에도 절대 사용 금지. 내용 길면 내용을 줄이거나 레이아웃 변경
  - 코딩 완료 후 강제 검증: fontSize grep → 52 미만 발견 시 즉시 수정
  - 동적 fontSize도 최솟값 52 이상 확인
- 설명은 키워드만 — 문장 전체를 화면에 넣지 않음
- 화법(고객에게 할 말)은 전체 문장 그대로 보여줌
- 부가 설명 텍스트 넣지 않음
- **두괄식 헤드라인**: 장면 시작 시 상단에 핵심 키워드 배치
  - 예외: [반전] 태그가 있을 때만 헤드라인 생략

## 색상 — 팔레트 시스템
- **매 영상 시작 전 5색 팔레트 확정**: 배경 / 카드 / 본문텍스트 / 보조텍스트 / 강조(1색)
- 전 장면 동일 팔레트 유지 — 팔레트 외 색 사용 금지
- 강조색은 영상 전체에서 1~2곳, 진짜 핵심만
- 변화 필요 시 같은 색의 명도(tint/shade)로 구분
- 원색 남발 금지

## 레이아웃
- **세로(9:16, 1080x1920)**: Safe Zone 상150 하170 좌우60. 위→아래
- **가로(16:9, 1920x1080)**: Safe Zone 상하80 좌우100. 좌→우
- 빈 공간은 요소를 키워서 채움 (억지 장식 X)
- 배경에 radial-gradient 깊이감
- **중앙축 정렬 철저** — 절대 규칙

## 애니메이션
- 의미 없는 반복 애니메이션 금지
- 등장 타이밍은 음성과 맞춤 — 음성보다 먼저 정보 노출 금지
- spring 기반 자연스러운 움직임
- 숫자 카운트업에 fontVariantNumeric: "tabular-nums"
- 장면 사이 0.7초 쉼, 전환은 TransitionSeries fade
- 마지막 장면 +2~3초(60~90프레임) 여유

## 장면 유형 (대본에 태그로 표시)
- [키워드 강조] — 핵심 단어 크게
- [비교] — 막대 차트, A vs B
- [흐름도] — 단계별 화살표 연결
- [카드 나열] — 항목 리스트
- [숫자 임팩트] — 카운트업
- [대화 UI] — 말풍선 (화법용)
- [반전] — 헤드라인 생략, 전환점에서 키워드 등장

## 장면 톤
- **정돈 톤** (정의, 화법, 데이터): 글자+박스+차트. 신뢰감, 공식 자료 느낌
- **가벼운 톤** (사례, 에피소드, 공감): 이모지+도형+움직임. 직관적
- 판단 기준: "외워야 하나?" → 정돈 / "느끼면 되나?" → 가벼운
- 정돈 톤 참고: `references/style-clean/` (컨테이너, 아이콘vs이모지, 텍스트 위계)

## 제작 참고
- 교안은 구조/맥락 이해용 — 도형/레이아웃/색상을 따라 그리지 않음
- 텍스트보다 움직임, 도형, 아이콘으로 개념 표현
- 폰트: Pretendard (CDN)
