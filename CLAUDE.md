# Remotion 영상 제작 원칙

## 파이프라인
영상 제작은 6단계 스킬로 자동 진행된다. "영상 만들자" → AI가 순차 실행.
개별 단계 점프: `/v0-intake` `/v1-concept` `/v2-script` `/v3-sample` `/v4-build` `/v5-qa` `/v6-render`
오케스트레이터: `/video` — 각 단계의 모델(opus/sonnet), 관문, 진행 표시 규칙 포함.

- Claude: 구체화, 제작, 검수 (producer + builder)

## 텍스트
- 최소 폰트 52px — 어떤 장면에도 절대 사용 금지. 내용 길면 내용을 줄이거나 레이아웃 변경
  - 코딩 완료 후 강제 검증: fontSize grep → 52 미만 발견 시 즉시 수정
  - 동적 fontSize도 최솟값 52 이상 확인
- 설명은 키워드만 — 문장 전체를 화면에 넣지 않음
- 화법(고객에게 할 말)은 전체 문장 그대로 보여줌
- 부가 설명 텍스트 넣지 않음
- **두괄식 헤드라인**: 장면 시작 시 상단에 핵심 키워드 배치
  - 예외: [반전] 태그가 있을 때만 헤드라인 생략

## 색상 — 팔레트 시스템
- **매 영상 시작 전 5색 팔레트 확정**: 배경 / 카드 / 본문텍스트 / 보조텍스트 / 강조(1색)
- 전 장면 동일 팔레트 유지 — 팔레트 외 색 사용 금지
- 강조색은 영상 전체에서 1~2곳, 진짜 핵심만
- 변화 필요 시 같은 색의 명도(tint/shade)로 구분
- 원색 남발 금지

## 레이아웃
- **세로(9:16, 1080x1920)**: Safe Zone 상150 하170 좌우60. 위→아래
- **가로(16:9, 1920x1080)**: Safe Zone 상하80 좌우100. 좌→우
- 빈 공간은 요소를 키워서 채움 (억지 장식 X)
- 배경에 radial-gradient 깊이감
- **중앙축 정렬 철저** — 절대 규칙
- **박스 안 텍스트 줄바꿈 방지** — 고정 width 박스에 텍스트 넣지 말 것. `whiteSpace: "nowrap"` + 적절한 padding으로 텍스트가 박스 크기를 결정하게. 텍스트가 길면 의도적 `{"\n"}`으로 자연스러운 위치에서 줄바꿈
- **문장 2개 이상 → 줄바꿈** — 마침표/물음표/느낌표 뒤에서 `{"\n"}`으로 끊기. 한 문장이 길면 의미 단위(조사/접속사 앞)에서 끊기

## 애니메이션
- 의미 없는 반복 애니메이션 금지
- 등장 타이밍은 음성과 맞춤 — 음성보다 먼저 정보 노출 금지
- spring 기반 자연스러운 움직임
- 숫자 카운트업에 fontVariantNumeric: "tabular-nums"
- 장면 사이 0.7초 쉼, 전환은 TransitionSeries fade
- 마지막 장면 +2~3초(60~90프레임) 여유
- **지연 등장 요소는 조건부 렌더링 금지** — `{frame >= X && <div>}` 쓰면 flex 레이아웃이 밀림. 대신 처음부터 렌더하고 opacity로 숨길 것
- **비주얼 리듬 규칙 (정적 화면 금지)**
  - 씬 시작 후 5초 이내 첫 비주얼 변화 필수
  - 어떤 시점에서든 10초 이상 비주얼 변화 없으면 금지
  - 20초 넘는 씬은 서브 페이즈로 분할하거나 씬 자체를 쪼갤 것
  - 씬 마지막 요소 등장 후 최소 2초(60프레임) 유지 — 나오자마자 씬 전환되면 안 됨. 필요 시 마지막 요소의 등장 타이밍을 앞당길 것

## BEATS 시스템 (음성-비주얼 싱크)
**모든 비주얼 요소(텍스트, 이모지, 도형, 차트)는 BEATS 상수로 타이밍 제어한다.**
- `frame - 20` 같은 감 기반 하드코딩 금지
- **beats 매핑은 Whisper 실측 타임스탬프에서만** — 추정값/감으로 적지 말 것. words.json에서 해당 단어의 start 값을 직접 참조
- V2에서 생성한 word-level 타임스탬프 → V4에서 `{에피소드}-beats.ts`로 변환
- 씬 컴포넌트는 반드시 beats 파일에서 import하여 사용
- 타이밍 수정 시 beats.ts 한 파일만 수정 → 전체 반영
- 허용 오차: ±3프레임(0.1초) 이내
- 파일 구조:
  ```
  src/{에피소드}/
  ├── {에피소드}-beats.ts       ← 타이밍 single source
  ├── {에피소드}Full.tsx        ← Sequence 배치 + Audio
  ├── Scene1_xxx.tsx            ← BEATS import
  └── SceneLayout.tsx
  ```

## pageTitle 규칙
- **맥락 라벨만** — 결론/핵심 멘트를 제목에 넣지 않음 (스포일러 금지)
- 좋은 예: "승객들의 반응", "지하철 안, 그 한마디", "왜?", "상담 첫 3초"
- 나쁜 예: "다들 쳐다봤어요", "귓구멍을 파라고 해요?", "답을 안 줬으니까"
- 원칙: 제목은 "지금 뭘 보고 있는지" 알려줌. 결론은 음성+비주얼이 전달

## 비주얼 카탈로그 (공유 비주얼 사전)
v1-concept에서 영상 성격에 맞는 카탈로그 조합을 확정하고, v2-script에서 장면별로 레퍼런스를 명시한다.
- **V1** (VisualCatalog) — 텍스트 애니메이션, 모션, 톤 비교 (21종)
- **V2** (VisualCatalogV2) — SVG 도형, 이모지 패턴, 블록 효과 (20종)
- **V5** (VisualCatalogV5) — 데이터 시각화: 도넛/라인/퍼널/레이더/타임라인/게이지 (8종)
- V3(전환), V4(분위기)는 참고용. 주력은 V1+V2+V5.

## 장면 톤
- **정돈 톤** (정의, 화법, 데이터): 글자+박스+차트. 신뢰감, 공식 자료 느낌
- **가벼운 톤** (사례, 에피소드, 공감): 이모지+도형+움직임. 직관적
- 판단 기준: "외워야 하나?" → 정돈 / "느끼면 되나?" → 가벼운
- 정돈 톤 참고: `references/style-clean/` (컨테이너, 아이콘vs이모지, 텍스트 위계)

## 레퍼런스 관리
1. 참고 자료 → `references/` 저장, `references/index.md`에 기록
2. 새 영상 시 `references/index.md` 먼저 확인
3. 피드백 → 이 CLAUDE.md 해당 섹션에 누적 반영
4. 원천 데이터는 mybrain 레포 (`~/Desktop/workspace/mybrain/`)

   - `output/` — 과거 결과물 (톤/스타일 참고용)
