# 비주얼 레이아웃 가이드 (16:9, 1920×1080)

> **용도**: 모든 장면 제작 시 비주얼 비중과 레이아웃 구조의 기준
> **출처**: style-clean/ 10장 + youtube-analysis/ 76프레임 + remotion-design-guide.md 종합
> **핵심 원칙**: "텍스트만 띄우지 않는다. 비주얼이 정보를 전달하고, 텍스트는 보조한다."

---

## 0. 공통 원칙

### 배경
- 기본 배경: `#0B1120` (거의 순흑) — 남색/네이비 톤 금지
- 배경 glow: accent 색을 배경에 깔지 않음. glow는 **키워드 등장 순간 0.5초 flash**로만
- ParticleField: 최대 opacity 0.08, 콘텐츠와 겹치지 않는 영역에만

### 비주얼과 텍스트
- **텍스트만 있는 화면 = 다시 생각**. 최소 1개 비주얼 앵커 필수
- "비주얼" = 아이콘, 이모지, 도형, 컨테이너, 차트, SVG, 화살표
- 장면 내용에 따라 비주얼이 주인공일 때도, 텍스트가 주인공일 때도 있음
- 숫자로 비율을 고정하지 않음 — 내용이 요구하는 만큼 자연스럽게

### 컨테이너 (카드) 사용 규칙
- **쓸 때**: 비교, 카드 나열, 흐름도, 데이터 — 구조적 정보
- **안 쓸 때**: 반전, 키워드 강조 — 감정 전달 장면
- 카드 배경: `rgba(255,255,255,0.06~0.08)`
- 카드 테두리: `1~2px solid rgba(255,255,255,0.10~0.25)`
- 카드 border-radius: `20~28px`
- 카드 내부 padding: `28~36px`

### 아이콘/이모지 역할
- **비주얼 앵커**: 화면에서 시선이 먼저 가는 요소. 텍스트보다 위 또는 좌측에 배치
- 크기: 최소 80px, 일반 120~160px, 히어로 180~200px
- 정돈 톤 → 라인 아이콘 (SVG circle+stroke) 선호
- 가벼운 톤 → 이모지 허용
- 라인 아이콘이 못생기면 이모지가 나음

### accent 색상 사용
- accent는 **테두리, 아이콘 stroke, 키워드 텍스트, 번호 뱃지**에만
- accent를 배경색으로 깔지 않음 (glow 포함)
- 영상 전체에서 1~2곳 진짜 핵심에만 사용

### 텍스트
- 화면에 올리는 텍스트는 **키워드/구 단위** — 문장 전체 금지 (화법 제외)
- 음성이 문장을 말해줌 → 화면은 핵심만 잡음
- 한 화면 텍스트 최대 2줄

## 1. [키워드 강조] 레이아웃

**용도**: 핵심 단어를 크게 보여주는 장면
**톤**: 정돈 또는 가벼운 모두 가능

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│         🎯 (이모지/아이콘)        │  ← 비주얼 앵커: 120~160px
│      ──────────────             │
│       핵심 키워드                 │  ← hero: 100~140px/900
│      ──────────────             │
│       보조 텍스트                 │  ← sub: 56~60px/600, 쿨그레이
│                                 │
└─────────────────────────────────┘
```

- **텍스트가 주인공, 아이콘이 앵커** 역할
- 이모지/아이콘이 키워드 위에 위치, 시선 유도
- WordHighlight로 키워드 배경 바 강조
- 나머지 요소 dim (60% 시점에서 opacity 0.4)
- **레퍼런스**: style-clean/6.png (단일 카드+아이콘+숫자), style-clean/11.png (시계+메시지)

## 2. [비교] 레이아웃

**용도**: A vs B, Before/After, 잘못 vs 올바른
**톤**: 정돈 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│   ┌──────────┐  ┌──────────┐   │
│   │  ✕ 아이콘  │  │  ○ 아이콘  │   │  ← 각 카드 상단에 아이콘
│   │          │  │          │   │
│   │ 키워드    │  │ 키워드    │   │  ← 72px/700
│   │ 보조설명  │  │ 보조설명  │   │  ← 56px/600
│   │          │  │          │   │
│   └──────────┘  └──────────┘   │
│        ◀ 중앙 구분선 ▶          │
│      [결론 한줄 or 프로그레스바]   │  ← accent, 하단
└─────────────────────────────────┘
```

- **카드 구조가 주인공** — 카드 자체가 시각적 구조를 전달
- 좌: wrongRed 테두리/아이콘, 우: rightGreen 테두리/아이콘
- 카드 사이 구분선: `1~2px, rgba(255,255,255,0.15)`
- 나중 카드 등장 시 이전 카드 dim
- 하단에 결론 키워드 or 프로그레스바
- **레퍼런스**: style-clean/8.png (좌우 카드+화살표+프로그레스), style-clean/9.png (좌우 대비)

## 3. [흐름도] 레이아웃

**용도**: 단계별 프로세스, 순서, 인과관계
**톤**: 정돈 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│  ┌──┐        ┌──┐        ┌──┐  │
│  │①│  ───▶  │②│  ───▶  │③│  │  ← 원형 아이콘 + 번호 뱃지
│  │아│        │아│        │아│  │
│  │이│        │이│        │이│  │
│  │콘│        │콘│        │콘│  │
│  └──┘        └──┘        └──┘  │
│  단계명      단계명       단계명  │  ← 64px/700
│  보조설명    보조설명     보조설명  │  ← 52px/600
│                                 │
└─────────────────────────────────┘
```

- **아이콘+화살표가 주인공**, 텍스트는 라벨 역할
- 원형 컨테이너 (stroke: accent, fill: transparent or 5% white)
- 화살표: SVG line + arrowhead, accent 색상
- 순차 등장 (spring stagger)
- 번호 뱃지: 작은 원형, accent 배경, 흰 텍스트
- **레퍼런스**: style-clean/5.png (3단계 원형 아이콘+화살표)

## 4. [카드 나열] 레이아웃

**용도**: 항목 리스트, 3~5개 포인트 정리
**톤**: 정돈 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│  ┌────────┐ ┌────────┐ ┌────────┐
│  │ 🔔     │ │ 👍     │ │ 📝     │  ← 이모지/아이콘 상단
│  │ 키워드  │ │ 키워드  │ │ 키워드  │  ← 64~72px/800
│  │ 보조   │ │ 보조   │ │ 보조   │  ← 52~56px/600
│  └────────┘ └────────┘ └────────┘
│                                 │
└─────────────────────────────────┘
```

- **카드+아이콘과 텍스트가 균형** — 어느 쪽도 압도하지 않음
- 카드 3~4개 가로 배치 (5개 이상이면 2행)
- 각 카드 상단에 이모지/아이콘 앵커
- 카드 내부: 아이콘(80~100px) → 키워드 → 보조 (세로 정렬)
- 순차 등장 (stagger 10~15프레임)
- **레퍼런스**: youtube-analysis/frame_030.jpg (카드 3개+이모지), style-clean/10.png (동심원+번호 콜아웃)

## 5. [숫자 임팩트] 레이아웃

**용도**: 핵심 수치를 크게 보여주는 장면
**톤**: 정돈 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│          ┌────────────┐         │
│          │  아이콘      │         │  ← 라인아이콘 or 도넛차트
│          │             │         │
│          │   10만원     │         │  ← 히어로 숫자: 140~180px/900, accent
│          │  보조 텍스트  │         │  ← 56~60px/600
│          └────────────┘         │
│                                 │
│      [프로그레스바 or 게이지]      │  ← 하단 보조 비주얼
└─────────────────────────────────┘
```

- **차트/게이지가 주인공**, 숫자가 핵심 포인트
- 숫자는 CountUp 애니메이션 + tabular-nums
- 도넛 차트, 프로그레스 바, 게이지 등 데이터 비주얼과 조합
- 숫자 등장 전 아이콘/차트 먼저 → 숫자 카운트업 → 강조
- **레퍼런스**: style-clean/7.png (도넛+숫자), style-clean/6.png (아이콘+숫자)

## 6. [대화 UI] 레이아웃

**용도**: 고객-FP 대화, 말풍선 인용
**톤**: 가벼운 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│           👤 "고객 말"            │  ← 좌측 정렬, 카드bg
│                                 │
│     "FP 말" 🧑‍💼                  │  ← 우측 정렬, accent 테두리
│                                 │
│      ┌─────────────────┐        │
│      │ 💡 핵심 인사이트   │        │  ← 중앙 카드, 아이콘+키워드
│      └─────────────────┘        │
│                                 │
└─────────────────────────────────┘
```

- **말풍선 카드 자체가 비주얼** 역할 — 별도 장식 불필요
- 말풍선은 **화면 너비 60% 이상** 차지 (작은 말풍선 금지)
- 고객: 좌측 정렬, 카드 배경
- FP: 우측 정렬, accent 테두리 (배경은 accent 10% 투명)
- 대화 아래 인사이트 카드: 아이콘 + 키워드 1줄
- 인사이트는 문장이 아닌 **"핵심 = 결론" 구조**
- 대화 버블 등장 시 이전 버블 dim
- **레퍼런스**: youtube-analysis/frame_001.jpg (채팅UI), youtube-analysis/frame_050.jpg (좌우 카드)

## 7. [반전] 레이아웃

**용도**: 감정 전환점, 클라이맥스
**톤**: 드라마틱

```
┌─────────────────────────────────┐
│  (헤더 생략 — hideHeader)        │
│                                 │
│                                 │
│       작은 질문 텍스트            │  ← 60~72px, 쿨그레이 (70% opacity)
│                                 │
│       부정/전환 텍스트            │  ← 80~88px, 화이트, bounce
│                                 │
│    ★ 핵심 키워드 (크게) ★         │  ← 100~120px, accent, glow flash
│                                 │
│       보조 텍스트                 │  ← 56px, 쿨그레이
│                                 │
└─────────────────────────────────┘
```

- **텍스트 크기 변화 자체가 비주얼** — 별도 아이콘/도형 없이 감정 전달
- **헤더 생략** (hideHeader: true) — 화면 전체를 감정에 집중
- 텍스트 크기 단계적 변화가 비주얼 효과를 대체
- 키워드 등장 시: shake(0.5초) + glow flash(0.5초 후 소멸) + 이전 텍스트 dim
- glow는 **지속하지 않음** — 순간 반짝 후 사라짐
- 쿨그레이 텍스트: `rgba(240,237,232,0.7)` (배경과 충분한 대비 확보)
- 컨테이너 없음, 배경 깨끗

## 8. [차트] 레이아웃

**용도**: 데이터 시각화 (도넛, 라인, 퍼널, 타임라인 등)
**톤**: 정돈 톤

```
┌─────────────────────────────────┐
│  header                         │
│  separator                      │
│  pageTitle                      │
│                                 │
│     ┌──────────────────────┐    │
│     │                      │    │
│     │    [차트 영역]         │    │  ← 콘텐츠 영역의 60~70% 차지
│     │                      │    │
│     └──────────────────────┘    │
│                                 │
│    핵심 수치  |  해석 키워드       │  ← accent 숫자 + 화이트 키워드
│                                 │
└─────────────────────────────────┘
```

- **차트가 주인공**, 텍스트는 해석 한 줄만
- 차트 종류는 Visual Catalog V5 참조
- 차트 아래에 핵심 수치 + 해석 키워드 1줄
- 차트 애니메이션: spring 기반 순차 그리기
- **레퍼런스**: style-clean/7.png (도넛), style-clean/12.png (타임라인), style-clean/13.png (산점도)
