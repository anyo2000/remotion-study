# 비주얼 경험 노트

> **이 문서는 규칙집이 아니다.** 과거 제작에서 배운 "이러면 잘 됐고, 이러면 망했다"의 기록이다.
> 장면을 설계할 때 참고하되, 내용이 요구하는 비주얼을 이 문서가 제한하면 안 된다.

---

## 잘 된 것

### 화면 활용
- **좌우 분할 레이아웃**이 화면 활용도가 높다 (예: 후킹와이 Scene10 BEFORE/AFTER, EP01 씬2 만나라3줄+이제아니죠)
- **이모지 산개**(전체 화면에 absolute로 배치)가 텍스트 나열보다 강렬하다 (예: 후킹와이 Scene3 📱, Scene5 👁️)
- **숫자+게이지 좌우 배치**가 숫자 임팩트를 잘 전달한다 (예: 후킹와이 Scene9 타이머)

### 비주얼 요소
- **이모지+키워드 조합**이 텍스트만보다 훨씬 낫다
- **카드/박스 비율은 내용에 맞게** — 짧은 텍스트에 가로로 긴 박스는 어색. 4:3 근처가 안정적
- **2×2 그리드 카드**는 카드끼리 여백을 충분히(40px+) 두고, 각 카드 안에 이모지+키워드를 세로 배치하면 깔끔
- **취소선, dim, shake** 같은 모션이 감정 전달에 효과적

### 타이밍
- **씬 시작은 음성 타임스탬프 절대값** 기반 (`T(9.9)`, `T(32.2)`) — duration 합산 방식은 싱크가 어긋남
- **GAP 프레임 불필요** — 음성 자체의 호흡 쉼이 자연스러운 간격

### 후킹와이에서 가장 인상적이었던 장면들
- Scene3_Subway: 📱 7개 화면 전체에 흩뿌리기 — 8유형 분류에 안 맞지만 가장 기억에 남음
- Scene5_EyeFocus: 👁️ 8개 + SVG 수렴선 — "모든 시선이 한 명에게"를 물리적으로 표현
- SampleScene1: 200px ❓ 이모지 + "파라고? vs 말라고?" — 반전 장면인데 이모지가 주인공
- SampleScene9: 원형 타이머 + 좌우 텍스트 — 숫자의 긴장감

## 망한 것

### 레이아웃
- **가운데에만 작게 모아놓고 좌우 여백 낭비** — contentPad가 너무 크면 콘텐츠가 좁아짐
- **텍스트가 줄바꿈되는 레이아웃** — whiteSpace: "nowrap" 확인 필요. 줄바꿈이 되면 레이아웃이 깨짐
- **좌우 분할인데 한쪽이 flex:1로 늘어나는 것** — 양쪽 모두 적절한 크기로 중앙 배치해야 균형이 맞음

### 비주얼
- **텍스트만 세로로 나열** — 화법을 직접 보여줘야 하는 경우 외에는 이모지/도형/차트를 섞어야 함
- **8유형 레이아웃 템플릿 따라 하기** — 내용이 형식을 결정해야 함. "[반전]이니까 아이콘 없음"은 틀렸음 (후킹와이 최고 장면이 반전+이모지)
- **가로로 늘어난 얇은 카드** — 내용이 짧으면 카드도 작게. 빈 공간은 카드 바깥에 여백으로

### 싱크
- **씬 시작을 duration 합산으로 계산** — 음성은 연속 재생인데 씬마다 패딩+갭을 더하면 누적 오차 발생
- **전환 갭(GAP) 프레임** — Sequence 사이 빈 프레임이 모자이크/깨짐 유발

## 장면 설계 질문 (비주얼 방향표 작성 시)

1. **이 장면에서 시청자가 느껴야 할 감정은?** (긴장/공감/깨달음/재미/신뢰)
2. **비주얼 주인공은 뭔가?** (이모지? 숫자? 텍스트? 차트? 움직임?)
3. **화면 전체를 잘 쓰고 있나?** (좌우 분할? 산개? 중앙 집중?)
4. **텍스트만 있는 건 아닌가?** (화법 인용 외에는 비주얼 요소 필수)
5. **줄바꿈 없이 들어가나?** (텍스트 길이 vs 레이아웃 폭 확인)
